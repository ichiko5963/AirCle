【タスク抽出ルール（厳守）】
============================================================
● **Kanban追記用タスク抽出では、いち（市岡）がするもののみを抽出すること。**
● **議事録の「アクションアイテム（タスク）」セクションには、全てのタスクを記載するが、Kanban追記用タスク抽出では、いち（市岡）がするもののみを抽出すること。**
● **タスクは最大5つまで**に絞り、**最も重要なタスクのみ**を抽出すること。
● **まとめられるタスクは必ずまとめる**こと。
  例：「AmazonでHDMIケーブル購入」と「USBケーブル購入」→「必要なケーブル類を購入（HDMI・USB）」
  例：「確定申告書類準備」と「税務署提出」→「確定申告の準備・提出」
● **タスクは必ず以下の形式で生成すること：**
  ```
  - [ ] #[議事録のタイトル] タスク内容（担当：いち／期限：YYYY-MM-DD／優先度：High・Medium・Low）
  ```
  例：議事録タイトルが `2025-11-15_日常管理MTG` の場合
  → `- [ ] #日常管理MTG タスク内容（担当：いち／期限：YYYY-MM-DD／優先度：High・Medium・Low）`
● 期限の形式は必ず `YYYY-MM-DD` 形式（例：2025-11-20）
● 優先度は High / Medium / Low のいずれか
● 期限が曖昧 → 「今週中」を自動提案し、日付は `date +%Y-%m-%d` で取得した日付から7日後を設定
● 最優先カテゴリ（税務・支払いなど）は High

【Kanbanファイルへの自動追記手順（必須実行）】
============================================================
**重要：議事録作成後、必ず以下の手順を実行すること**

1. **議事録タイトルの取得**
   - 生成した議事録のタイトル（`# [日付]_[議事録の種類]`）を取得
   - 例：`# 2025-11-15_日常管理MTG` → プロジェクト名は `日常管理MTG`

2. **Kanbanファイルの読み込み**
   - `read_file` で以下のファイルを読み込む：
     `/Users/ichiokanaoto/Downloads/Ichioka Obsidian/07-To Do/Project_Kanban.md.md`

3. **タスクの追記**
   - 抽出したタスクを、Kanbanファイルの「## 日常」セクションの下に追記する
   - 追記する際は、既存のタスクの下に追加する
   - タスク形式は必ず `- [ ] #[議事録のタイトル] タスク内容（担当：いち／期限：YYYY-MM-DD／優先度：High・Medium・Low）` とする
   - 議事録タイトルから日付部分（`YYYY-MM-DD_`）を除いた部分をプロジェクト名として使用
   - 例：`# 2025-11-15_日常管理MTG` → `#日常管理MTG`

4. **ファイルの保存**
   - `search_replace` または `write` を使用して、追記した内容をKanbanファイルに保存する

5. **Kanbanファイル内のタスクを期限順に並び替える（必須実行）**
   - Kanbanファイル内の「## 日常」セクションのタスクを、期限（YYYY-MM-DD）の昇順で並び替える
   - 期限が早いタスクが上に来るようにする
   - `search_replace` または `write` を使用して、並び替えた内容をKanbanファイルに保存する

6. **確認**
   - 必要に応じて `read_file` で追記内容と並び順を確認する

**この手順を実行しないと、議事録作成タスクは完了していない扱いとみなします。**
