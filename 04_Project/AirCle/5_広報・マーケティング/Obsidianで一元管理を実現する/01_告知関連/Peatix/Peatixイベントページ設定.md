Obsidianで一元管理を実現する

### イベント説明
◎概要
「情報が散在していて、どこに何があるか分からない...」
「情報を集めるだけで、活用までつなげられていない...」
「複数のツールを使い分けているが、統合できていない...」

そんな悩みを抱えていませんか？

実は、情報が複数のツールに散らばっていて、一元管理ができていない――そんな経験はありませんか？

このセミナーでは、Obsidianを活用した情報の一元管理術をテーマに、個人ワークフローとチームワークフローの2軸で具体的な運用例を紹介します。特に「情報収集 × AI活用」を中心とした、Obsidianを核としたドキュメント管理フローを体系的に解説します。

一次情報をObsidianへ自動集約し、AIによる要約・構造化へつなげる。外部ドキュメント（Notion、Google Meet）を安全にObsidianへ取り込む。AI活用に最適化されたフォルダ構成で、情報の鮮度管理やAI活用時の効率化を実現する。

「情報散在」から「一元管理」へ。AI時代の情報管理を支える、統合ワークフロー設計を1時間で学べます。

◎ セミナー内容
### 【第1部】個人ワークフロー：一次情報を Obsidian に自動集約
- 一次情報の取り込み方法（Kindle Highlights Importer、Local RSS、Line Touch Ink、Obsidian Web Clipper）
- AIによる要約・構造化の実践
- 収集した一次情報をAIで要約する方法
- 構造化されたノートへの変換プロセス

### 【第2部】チームワークフロー：外部ドキュメントの読み取り専用統合
- Notion API × GitHub Actions での自動同期
- 初回インポート：Obsidian Importerの使い方

- Google Meetログの取り込み（議事録・文字起こしをObsidianへ取り込む方法）

- AI Output（AI生成物の格納先）
- Archive（古い情報の退避先）
- Notes（通常ノート）

### 【第4部】GitHub を活用した自動化（概要）
- Notion → Obsidian の同期
- Google Meet → Obsidian の同期
- RSSおよびメモの自動取得
- GitHub Issue を活用した事件管理

### 【第5部】AIルール設定（Cursor）

- ノート整形ルール
- カテゴリ分類ルール
- 情報フィルタリング
- Cursor ワークスペース構成方法

### 【第6部】まとめと質疑応答
