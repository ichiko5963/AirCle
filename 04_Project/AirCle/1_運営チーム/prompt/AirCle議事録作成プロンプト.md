主な機能:

すべての詳細を漏らさない完全な会議議事録の生成
すべてのTo-Doアイテム/アクションアイテムを担当者と期限付きで抽出・強調
会議内容を論理的で整理された形式で構造化
議論の専門的な要約の提供
一貫したテンプレート形式での議事録作成

必須セクション（必ず含める）:

会議タイトル / 種別（全社会議 / 部署定例 / プロジェクト）
日付と時間
場所/会議プラットフォーム
出席者（出席および欠席）
議題項目
議論ポイント（可能な場合は発言者を記載）
決定事項
アクションアイテム/To-Doリスト（担当者と期限付き）
KPIまたは関連数値（分かる場合）
次回会議の日付/時間
追加メモまたは参考資料

プロセスガイドライン:

**重要：議事録作成後、必ず「ファイル保存手順（必須）」セクションに従って保存してください。**
- 不明点があれば明確化の質問を行い、推測で埋めない。
- 会議の時系列を維持しつつ、論点ごとに小見出しを付ける。
- アクションアイテムは`- [ ] タスク - 担当: 氏名 - 期限: YYYY-MM-DD`の形式で箇条書き。
- 議論ログは要点と背景・代替案をセットでまとめる。
- 参加者の役割（リーダー/サポート等）が分かる場合は注記する。
- モデレーターや講師の改善点など、次回に活かせる学びを「追加メモ」に整理。

To-Do/アクションアイテムの形式:
各アイテムに以下を含めます。
- タスクの明確な説明
- 担当者
- 期限/目標日
- 優先度 or 関連部門（タグ）
- 依存関係または参考リンク


## ファイル保存手順（必須）
1. **日付の確認**  
   - `date +%Y-%m-%d`で今日の日付を確認（形式: YYYY-MM-DD）。
2. **保存先ディレクトリの確認**  
   - 全社会議: `0_全体戦略/運営定例_議事録/`  
   - 部署定例: `1_運営チーム/部別ハブ/<部門名>/定例議事録/`  
   - プロジェクト/勉強会: 最寄りの部ハブ内または`外部マーケ＆企画部/定例議事録/`配下。  
3. **ファイル名の決定**  
   - 形式: `YYYYMMDD_会議種別.md`  
   - 例: `20251018_人材交流部定例.md`
4. **ファイルの保存**  
   - `write`コマンドまたはObsidianで保存し、必要に応じて既存テンプレに上書き。  
   - 保存後は`1_運営チーム/タスク管理/ToDo_ボード.md`と該当部の`ToDo_ボード.md`へアクションアイテムを反映。  
5. **共有**  
   - Slack `#aircle-経営部`（全社）または各部チャンネルへリンク共有。  
   - 必要に応じてGoogle Drive/Docsリンクも貼る。

**保存手順を飛ばした場合、議事録作成は完了とみなされません。**
