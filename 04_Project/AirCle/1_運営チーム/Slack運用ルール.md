# Slack運用ルール
> 「Aircleはあったかい」と初見で伝わること、かつタスクが迷子にならないことが基準です。

## 1. リアクション&コメント基本
- 新規投稿/自己紹介には**30分以内に最低1リアクション＋1コメント**。人材交流部が率先してTop3リアクション数を独占する。  
- 交流会/説明会告知は`🔥` or `🎉`、タスク完了報告は`✅`で統一。  
- 週次でRyucleレポートを確認し、リアクション数が落ちたチャンネルを重点対応する。  

## 2. チャンネル使い分け
| チャンネル | 目的 | 投稿フォーマット |
| --- | --- | --- |
| `#aircle-general` | 全体アナウンス/議事録共有 | `[YYYY/MM/DD] 件名`→本文→`📎`資料リンク |
| `#aircle-経営部` | 意思決定・タスク確認 | `決定事項 / 依頼 / 期限`の3行固定 |
| `#aircle-ops-todo` | タスクハンドオフ | `- [ ] タスク (担当 @name | 期限)`形式 |
| `#aircle-onboarding` | 新規メンバー歓迎・案内 | テンプレ挨拶＋交流会/1on1リンク |
| 各部チャンネル | 部内共有・ラフな相談 | `#部名-ops`で統一。タグ付けは@部リーダーへ。 |

## 3. オンボーディング導線
1. `#aircle-onboarding`で歓迎 → 人材交流部がリアクション。  
2. 24h以内に人材交流部が1on1の日程候補をDM。  
3. 面談議事録リンクをNotion/Obsidianに保存し、`#aircle-経営部`に共有。  
4. 配属決定後は該当部の`ToDo_ボード.md`に名前を追記し、Slackの役割タグを更新。  

## 4. 投稿テンプレ
```
[件名]（例：11/24 Obsidian講座 告知第2弾）
目的: 何を達成したいか（例：学生申込30名→50名）
本文: 3行以内で概要
タスク: - [ ] 担当 @name | 期限
リンク: Peatix / notion / Drive
```

## 5. DM / 1on1 エチケット
- DMを送る場合はまずチャンネルで声掛け→反応がなければDM。  
- 面談テンプレに沿って議事録を残し、担当リーダーへタグ付け。  
- 返信がない場合は24h後にGentle Reminder、48h後に電話/別経路で確認。  

## 6. 緊急時
- 30分以内に対応が必要な場合は`@channel`ではなく、`@here`＋電話。  
- Peatix/配信トラブルは`#aircle-emergency`（通話専用）に集約。  

Slack文化を守ることが最速の採用とリテンションに繋がります。全員が「最初のリアクション担当」という意識で運用してください。
